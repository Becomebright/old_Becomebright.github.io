零崎的战争

时间限制: 1500 ms 内存限制: 65536 kb

总通过人数: 161 总提交人数: 172

题目描述

不知道各位听没听说过《This War of Mine》这个游戏，不过没听说过也没有关系。简单来说，这是一个生存游戏，作为一个被战争波及到的无辜平民，你需要在战争中探索周边地区获得补给从而活下去。

现在无聊的零崎有这样一个设想，如果这把这个游戏的地图挂在Google Map上，以真实地图为背景做成一个网络游戏可能会很好玩。假设世界上所有的城市已经被分为玩家初始所在地和可探索区域两类，玩家自由选择一个初始所在地，那么作为一个机智的玩家，你应该选择哪里才能让自己被FTK(首轮击杀)的可能性最小呢？

输入

第一个数为测试轮数K。

每轮测试以N开始，下面N行，代表初始地点，每行两个整数X Y表示其坐标。接下来N行代表可探索区域，每行两个整数X Y表示其坐标。

1 ≤ N ≤ 100000. 0 ≤ X，Y ≤ 1000000000

输出

每轮测试一行，输出为初始地点和可探索区域间的最近距离，结果保留小数点后三位。

解题思路

本题题意为求两个点集之间的最小距离。

我们先将问题简化，求一个点集内部的最近点对。

简单的想法是两两遍历，复杂度为O(n^2)，对于此题，n ≤ 10^5，显然是无法接受的。

更好的做法是采用分治，先将点集以x为关键字排序，再以x为关键字进行二分,，即将[l, r]分成[l, m]与[m+1, r]，并在分开的两个点集中分别寻找最近点对，求得最小距离d1,d2，从而得到总最小距离d = min(d1, d2)。然而最近点对可能不同时处于一个点集（一个点集中有一个），面对这种情况，有一个很好的性质：最近点对的x坐标一定处在区间[m.x-d, m.x+d]范围内，此时只需要两两遍历该区间内的点即可。

此外需要注意的是分治的极端情况，由于分治的最小单位是点对，也就是两个点，所以当区间内只有两个点时，必是最近点对。除此以外，当区间内只有三个点，也不能继续分割，否则会产生一个点，此时应在三个点中直接寻找最近点对。

再回到原始问题，两个点集的情况需要对每个点加一个标记，以判断属于哪个点集，在进行分治处理时注意判断点对是否分属不同点集即可。l

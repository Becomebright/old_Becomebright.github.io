Problem Description

Magry最近看到了一个问题——

给定四个等长整数集合A,B,C,DA,B,C,D， 要求分别从中选取一个元素 a,b,c,da,b,c,d 使得 a+b+c+d=0a+b+c+d=0 成立。

问：有多少种选法？

乍一看这问题挺简单的，他觉得完成它是一件轻而易举的事。

可是一看这计算量，他被吓成一团了……

Input

多组测试数据（不超过5组），以EOF结尾。

每组测试数据第一行为一个正整数n，表示四个数组的长度。0<n≤4000

Output

对于每组数据，输出一行，一个数，选择方法总数。

Sample Explanation

很显然，样例当中只有 1+4−1−4=01+4−1−4=0 和 2+3−1−4=02+3−1−4=0 这两种情况成立。

解题思路

很容易想到分别枚举四个数组的四个数，看相加是否为0，然而这样的复杂度是O(n^4)。

换一种思路，因为−500000≤x≤500000，两个数相加则范围是−1000000≤x≤1000000，总共有2000001种情况，我们可以将两个数组中的元素两两相加，合并成一个数组，统计每种情况的个数，再统计能够相加得到0的种数。

语文不好，感觉说的不清楚，拿样例说一下：

两个数组合并得到：

这一步的复杂度是O(n^2)

那么第一个数组中，4出现1次，5出现2次，6出现1次

第二个数组中，-91，-5，0，91各出现一次

只有-5和5相加能得到0，则能得到0的情况共2*1=2次。

可以用数组b[1000000 + i]统计第一个数组中i出现的次数（之所以加1000000是因为数组下标不能为负），那么对于第二个数组中的每个数j，我们只需在O(1)的时间内查找b[1000000 - j]的值就能得到相加得0的种数。

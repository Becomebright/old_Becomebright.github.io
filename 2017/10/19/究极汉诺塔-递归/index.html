题目描述

汉诺塔大家都清楚的啦，普通的汉诺塔肯定难不倒大家啦，于是王木木出了一道究极汉诺塔的题，做完这题，你就可以大声说——

究极汉诺塔不知道比别的汉诺塔高到哪里去了，我和它谈笑风生

如图，给定初始局面和目标局面，求最少步数。

难题慎入

输入

多组测试数据

第一个数为数据组数n （1<=n<=60）

第二行包含n个1~3的整数，第i个数也就是初始局面中的第i个盘子所在的柱子编号（1号盘子最小，n号盘子最大）

第三行与第二行格式相同，为目标局面。

当n位0时，输入结束

输出

对于每组数据，输出一行，为最小步数的值

输出样例

提示

普通的汉诺塔的步数是(1<<n)-1

解题思路

解决本题需要用到以下几个结论：

  1. 普通的汉诺塔问题的步数是(1<<n)-1，即将n个盘子从一个柱子移动到另一个柱子的步数是(1<<n)-1
  2. 汉诺塔从状态A转移到状态B的最小步数，与从状态B转移到状态A的最小步数相同。

现在来考虑实际的问题，汉诺塔问题的处理原则是先大后小。先考虑最大的一块盘子，想要搬动最大的一块，需要将这一块上面的盘子和目标位置上的盘子移到辅助柱上。设这些盘子都在辅助柱上的状态称为C，初始状态为S，目标状态为D，整个过程分为3步：

  1. S转移至C
  2. 将最大的盘子移动至目标柱
  3. C转移至D（利用结论2，可变为为“D转移至C”）

设计一个函数long long Move(int stat[], int bottom, int support)，表示从当前状态stat，将bottom及更小的盘子都移动到辅助柱support，返回总步数。则上述三步的步数分别为：

  1. Move(S, bottom-1, support)
  2. 1
  3. Move(D, bottom-1, support)

对于函数Move(stat, bottom, support)，实现方式为：

从大到小检查stat状态是否有盘子不在support柱，将其记为bottom

  1. 若bottom==0, 说明 不需要移动盘子，返回0

  2. 否则，需要先将bottom以上的盘子移动到辅助柱（注意是新的辅助柱，不是support），接着将bottom盘移动到support柱，最后将辅助柱上的bottom-1个盘子移动到support柱。

    总步数为Move(stat, bottom-1, 6-stat[bottom]-support) + 1 + Hanoi(bottom-1)
